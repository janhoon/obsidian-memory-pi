# Obsidian Memory Wiki

This vault is a maintained knowledge base for Pi.

## Rules

- `sources/` is immutable source material and generated ingest artifacts
- `memory/` is the maintained wiki
- use `memory_ingest_source` for local-path or URL documents/images/videos that should enter memory
- search first, read sparingly, synthesize before writing
- use the wiki to answer continuity, decision, and project-context questions
- when durable context changes, update the relevant memory notes and `memory/log.md`

## Retrieval order

1. `memory/schema.md`
2. `memory/index.md`
3. `memory/working-context.md`
4. project-specific notes under `memory/projects/`
5. session notes under `memory/sessions/` when recent context matters
